启动容器（后台模式）

使用-d参数创建一个以进程方式运行的容器

这时我们并没有直接进入容器的终端中，取而代之的是输出了一串长字符串，这个字符串是容器ID，对于每个容器都是唯一的，我们可以通过容器ID来指定某一容器。刚刚启动的容器以后台模式运行，可以使用docker ps命令查看当前运行中的容器

输出说明：

CONTAINER ID: 容器 ID。

IMAGE: 使用的镜像。

COMMAND: 启动容器时运行的命令。

CREATED: 容器的创建时间。

STATUS: 容器状态。

状态有7种：

  • created（已创建）
  • restarting（重启中）
  • running 或 Up（运行中）
  • removing（迁移中）
  • paused（暂停）
  • exited（停止）
  • dead（死亡）

PORTS: 容器的端口信息和使用的连接类型（tcp\udp）。

NAMES: 自动分配的容器名称。

在主机中使用docker logs <CONTAINER ID/NAMES>查看容器内的标准输出

停止容器

使用docker stop <CONTAINER ID\NAMES>命令来停止容器

这时再使用docker ps查看发现刚才停止的容器已经不见了

进入容器

使用-d参数创建容器的时候，并不会直接进入容器，会在后台运行，如果想要进入容器，可以使用docker attach或docker exec命令

docker attach与docker exec的区别：

docker attach在退出容器时会将容器停止，docker exec不会停止容器

docker镜像使用

列出本地镜像列表

使用docker images命令可以列出本地的镜像列表

输出说明:

  • REPOSITORY：表示镜像的仓库源
  • TAG：镜像的标签

同一个仓库源中可以有多个不同的标签，用来代表不同的版本，例如同一个仓库源ubuntu，有15.10、lastest两个版本，我们使用REPOSITORY:TAG来定义一个镜像

  • IMAGE ID：镜像ID
  • CREATED：镜像创建时间
  • SIZE：镜像大小

查找镜像

我们可以直接在Docker Hub网站上查找镜像，网址为：https://hub.docker.com/

也可以使用docker search命令搜索所需要的镜像，例如我现在需要使用httpd镜像来搭建web服务，可以使用下面的命令来搜索

输出说明：

  • NAME: 镜像仓库源的名称

  • DESCRIPTION: 镜像的描述

  • OFFICIAL: 是否 docker 官方发布

  • stars: 类似 Github 里面的 star，表示点赞、喜欢的意思。

  • AUTOMATED: 自动构建。

获取一个新的镜像

当我们使用一个本地不存在的镜像时，docker会自动下载所需要的镜像，如果我们想预先下载好镜像，可以使用docker pull命令来下载

把镜像拉取下来后，就可以直接使用该镜像而无需从远端下载

删除镜像

删除一个镜像时需要先使用docker rm命令删除所有依赖于这个镜像的容器

删除镜像使用docker rmi命令，例如删除ubuntu:15.10镜像

创建镜像

当docker镜像仓库中的镜像已经不能满足需求时，我们可以利用以下两种方式自己对镜像进行更改：

  1. 从已经创建的容器中更新镜像，并且提交这个镜像
  2. 使用Dockerfile新建一个镜像
